Terrakettle

Terrakettle is the companion web service to Terrahawk — where Terrahawk reports come to rest. Terrahawk runs terragrunt plan across your infrastructure and emits a static HTML report per run; Terrakettle stores those reports per project, keeps the full history, and serves the original interactive report from a browser — adding status badges, run-to-run comparison, feeds, metrics, and drift notifications on top.

How it works

Terrahawk pushes each scan to Terrakettle's authenticated endpoint (--push-url / --push-token). Report payloads (HTML, _data.js, JSON) are written to object storage (local disk, S3, Azure Blob, or GCS); a small SQLite index holds the per-run metadata. The web UI never alters the stored report — it streams Terrahawk's HTML verbatim and wraps it in history, search, and access control.

Features

🗂️ Per-project history — every run filed under a project, newest-first, with clean/drift/error/timeout counts, a drift trend sparkline, status filtering, and pagination.

📤 Push API — a single token-authenticated endpoint Terrahawk uploads to after each scan. Per-project bearer tokens (optional expiry + rate limit), guarded by an admin key.

🗄️ Object storage — report files live in S3 / Azure Blob / GCS (or local disk); large sidecars redirect to short-lived presigned URLs so the object store serves the bytes directly.

🦅 Original report, untouched — the interactive Terrahawk HTML (diagrams, diffs, CSV export) is served exactly as generated; its _data.js loads as a sibling.

↔️ Run compare — diff any two runs to see which units flipped clean → drift / error over time.

🛡️ Status badge — an embeddable SVG shield per project (/p/<slug>/badge.svg), public even when viewing is gated.

📡 Feeds & metrics — JSON + RSS run feeds, Prometheus /metrics, and a deep /healthz that probes the DB and storage.

🔔 Notifications — fire a Slack / Teams / generic webhook when a pushed run has drift or errors.

🔒 Access control — optional password-gated viewing (signed session cookie); reports served with a strict CSP. Refuses to start on the default admin key.

🎨 Dark / Light theme — toggle with the moon button in the header; the preference is saved in localStorage and persists across sessions.
